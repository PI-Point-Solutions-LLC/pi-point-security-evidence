# Security Control Ownership

Reviewed: 2026-07-14

| Role | Accountable owner | Review cadence |
| --- | --- | --- |
| Security program and internal audit | PI Point owner | Quarterly and after material change |
| Incident lead | PI Point owner | After every incident/tabletop |
| Release approval | PI Point owner | Each production release |
| Data/privacy owner | PI Point owner | Quarterly and after data-flow change |
| Vendor-risk owner | PI Point owner | Quarterly for critical vendors |
| Privileged-access review | PI Point owner | Quarterly |
| Database/Edge Function security | PI Point owner | On change and quarterly evidence review |
| Smokeball integration security | PI Point owner | On change, incident and quarterly evidence review |
| Communications | PI Point owner, with legal/support assistance if retained | Incident/tabletop and annual contact review |

Small-organization role consolidation is explicit. Objectivity safeguards are described in the [internal audit function](internal-audit-function.md).
